# Real User Checklist

## TL;DR

Do these 4 checks before May 9:

1. Real iPhone Safari login + tap-through
2. Real booking + cancel
3. Real event RSVP
4. Real admin announcement to:
   - one opted-in member
   - one opted-out member

If those feel normal, launch confidence is high.

## ELI5

Automated tests are practice.
Real users are real life.

This checklist is just making sure the app feels normal on a real phone with real accounts.

## Checks

### 1. iPhone Safari

- Log in on a real iPhone using Safari
- Confirm it opens the right app/experience
- Tap through:
  - Home
  - Book
  - Schedule
  - Messages
  - Settings
- Open an event or RSVP modal

Pass if:
- no weird jumps
- no clipped buttons
- no broken nav
- no odd Safari layout issues

### 2. Booking

- Book a court as a real member
- Confirm booking appears correctly
- Cancel it

Pass if:
- booking saves
- cancellation works
- UI updates correctly

### 3. Event RSVP

- Open a real event
- Tap RSVP
- Refresh/check again

Pass if:
- RSVP sticks
- no duplicate/confusing behavior

### 4. Admin Announcement

- Send an announcement from admin
- Use:
  - one member with announcements ON
  - one member with announcements OFF

Pass if:
- opted-in member gets announcement
- opted-out member does not

## Done Means

You're good if:

- everything works
- nothing feels confusing
- no one gets the wrong notification
